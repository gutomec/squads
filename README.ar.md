# أنت تدير جيشاً من فرد واحد. والنتائج واضحة.

عميل ذكاء اصطناعي واحد يفعل كل شيء. يكتب الكود، يراجع الكود، يختبر الكود، ينشر الكود، يخطط للبنية، يدير المتأخرات، يتولى DevOps.

هذا ليس نظاماً. هذه فوضى مُغلَّفة بـ prompt.

تعرف ماذا يحدث حين يتولى عميل واحد عشرة وظائف؟ نفس ما يحدث حين يتولى موظف واحد عشرة وظائف. كل شيء يُنجَز. لا شيء يُنجَز بشكل صحيح.

---

## المشكلة الحقيقية في برمجة الذكاء الاصطناعي اليوم

الجميع يتحدث عن عملاء الذكاء الاصطناعي. لا أحد يتحدث عن طريقة استخدامهم.

إليك ما يفعله معظم الناس: يفتحون Claude Code، يعطونه prompt ضخماً، ويأملون. يكتب العميل الكود، يراجع كوده بنفسه، يختبر اختباراته بنفسه، وينشر فوضاه بنفسه. لا فصل في المسؤوليات. لا حدود. لا محاسبة.

هذا مثل توظيف شخص واحد ليكون الرئيس التنفيذي، والمحاسب، والعامل، ورئيس الأمن. في نفس اليوم. بنفس الراتب.

يعمل حتى لا يعمل. وحين يتوقف، لا تعرف أي "دور" تسبب في المشكلة، لأنه لم يكن هناك دور حقيقي أصلاً.

## Squads تحل هذا

Squad هو فريق. ليس عميلاً واحداً يتظاهر بأنه فريق. مجموعة منظمة فعلية من العملاء، كل واحد منهم له اسم، ووظيفة، وحدود، ومهام يمتلكها.

فكر في الأمر هكذا: بدلاً من ذكاء اصطناعي واحد يفعل كل شيء، تبني فريقاً حيث كل عميل متخصص.

- كاتب يكتب فقط
- مراجع يراجع فقط
- منسق يُنسِّق
- مدقق يتحقق من الجودة

كل عميل له ملف markdown خاص به يحتوي على شخصيته، وأوامره، وقواعده. كل مهمة لها شروط مسبقة وشروط لاحقة. كل workflow يُحدد كيف يتعاون العملاء — من يبدأ، من يراجع، من يوافق.

لا تُضبط هذه الأمور على منصة احتكارية. لا تدفع مقابل مقاعد. لا تتعلم أداة جديدة.

إنه مجرد مجلد. في مشروعك. يحتوي على ملفات markdown.

```
squads/my-squad/
├── squad.yaml          # البيان — من في هذا الفريق
├── agents/             # ملف .md واحد لكل عميل
├── tasks/              # ملف .md واحد لكل مهمة
├── workflows/          # أنماط التعاون
├── config/             # إعدادات على مستوى الـ squad
└── README.md           # التوثيق
```

انسخ المجلد إلى مشروع آخر. يأتي الـ squad معك. بدون تثبيت، بدون ترحيل إعدادات، بدون مفاتيح API. العملاء والمهام والـ workflows كلها موجودة في المجلد.

هذا هو قابلية النقل الحقيقية. ليست "صدِّر إلى JSON وتأمل" قابلية نقل.

## كيف يعمل فعلياً

Squads هي skill لـ [Claude Code](https://docs.anthropic.com/en/docs/claude-code). تثبتها مرة واحدة. ثم تبني الفرق.

### التثبيت (30 ثانية)

```bash
cp -r squads-skill/ your-project/.claude/skills/squads/
mkdir -p your-project/squads/
```

تم. بدون npm install. بدون pip install. بدون Docker. بدون جحيم YAML. تستخدم الـ skill أدوات Claude Code الأصلية — Read, Write, Edit, Glob, Grep, Bash. لا شيء خارجي.

### إنشاء أول squad

```bash
/squads *create-squad content-pipeline
```

هذا ينشئ هيكل المجلد الكامل، ويولد بيان squad.yaml، ويضبط الإعدادات. أصبح لديك الآن فريق فارغ. ابدأ بإضافة العملاء.

```bash
/squads *add-agent content-pipeline writer
/squads *add-agent content-pipeline reviewer
/squads *add-agent content-pipeline publisher
```

ثلاثة عملاء. كل واحد يحصل على ملف .md خاص به بشخصيته، وتعريف خبرته، وأوامره. الكاتب يكتب. المراجع يراجع. الناشر ينشر. لا أحد يؤدي عمل الآخر.

### التسجيل للـ slash commands

```bash
/squads *register-squad content-pipeline
```

الآن يمكنك استدعاء أي عميل مباشرة:

```bash
/SQUADS:content-pipeline:cp-writer
/SQUADS:content-pipeline:cp-reviewer
```

بدون إعداد إضافي. بدون منطق توجيه. سجِّل مرة، استخدم للأبد.

## 7 أشياء يمكنك فعلها

| الأمر | ماذا يفعل |
|---------|-------------|
| `*create-squad {name}` | ينشئ هيكل squad كامل |
| `*list-squads` | يعرض جميع الـ squads مع عدد العملاء |
| `*add-agent {squad} {role}` | يضيف متخصصاً إلى الفريق |
| `*validate-squad {name}` | يشغِّل 20 فحصاً للتكامل |
| `*register-squad {name}` | يفعِّل slash commands للـ squad |
| `*install-squad-deps {name}` | يثبت تبعيات Node (pnpm) و Python (uv) |
| `*run-workflow {squad} {name}` | ينفِّذ workflow للتعاون |

هناك المزيد من الأوامر. هذه هي التي ستستخدمها كل يوم.

## الـ Workflows — كيف يتعاون العملاء فعلياً

العميل وحده مفيد. العملاء الذين يتعاونون خطيرون (بالمعنى الإيجابي).

5 أنماط جاهزة:

**Pipeline** — العميل A ينتج مخرجات، العميل B يحولها، العميل C ينهيها. تسلسلي. مثل خط التجميع. استخدمه لإنتاج المحتوى، ومعالجة البيانات، وأي شيء له مراحل واضحة.

**Hub-and-Spoke** — منسق واحد، متخصصون متعددون. المنسق يفوِّض، يجمع النتائج، يلخِّص. استخدمه حين تحتاج إلى منسِّق.

**Review** — عميل يؤدي عملاً، مراجع يفحصه، يعود إذا فشل. نمط بوابة الجودة. لا شيء يُشحَن حتى يوافق المراجع. استخدمه لأي شيء يجب أن يكون صحيحاً.

**Parallel** — عملاء متعددون يعملون على مهام مستقلة في نفس الوقت، ثم يدمجون النتائج. استخدمه حين لا تعتمد المهام على بعضها.

**Teams** — تنسيق في الوقت الفعلي عبر Claude Code Agent Teams. العملاء يتواصلون أثناء التنفيذ. استخدمه للعمل المعقد متعدد الخطوات حيث يحتاج العملاء للتفاعل مع بعضهم.

اختر النمط المناسب. أو ادمجهم. pipeline حيث كل خطوة لها حلقة review. hub-and-spoke حيث يعمل المتخصصون بالتوازي. اخلطهم ودمِّجهم.

## صفر تلوث للسياق

هذا الجزء الذي لا يتقنه أحد غيرنا.

معظم إعدادات multi-agent تحمِّل كل تعريفات العملاء في نافذة السياق عند البدء. 10 عملاء؟ هذا 10 ملفات markdown تأكل tokens قبل أن تطرح سؤالاً واحداً.

Squads لا تحمِّل شيئاً حتى تطلبه.

`*create-squad` يحمِّل بروتوكول الإنشاء. `*validate-squad` يحمِّل قائمة التحقق. `/SQUADS:my-squad:my-agent` يحمِّل ملف .md لذلك العميل وحده. لا شيء آخر.

10 squads في مشروعك؟ 100 squad؟ صفر tokens إضافية في نافذة سياقك حتى تستدعي واحداً. الـ skill تقرأ بدقة جراحية — فقط ملف المرجع الذي تحتاجه، في اللحظة التي تحتاجه فيها.

نافذة سياقك تبقى نظيفة. دائماً.

## التبعيات بدون صداع

Squads يدعم 7 أنواع من التبعيات. التثبيت كسول — لا شيء يُثبَّت حتى تقول ذلك.

| النوع | المدير | الحالة |
|------|---------|--------|
| Node | pnpm | نشط |
| Python | uv | نشط |
| System | — | توثيق فقط |
| Squads (cross-squad) | — | نشط |
| MCP tools | — | توثيق فقط |
| Go | go modules | محجوز |
| Rust | cargo | محجوز |

كل squad يحتفظ بـ `node_modules/` و `.venv/` الخاصة به. لا تعارضات بين الـ squads. لا تلوث عالمي.

```bash
/squads *install-squad-deps my-squad    # يثبت كل شيء
/squads *check-squad-deps my-squad      # يفحص بدون تثبيت
```

## التحقق — 20 فحصاً قبل الشحن

```bash
/squads *validate-squad my-squad
```

9 فحوصات إلزامية يجب أن تجتاز: squad.yaml صالح، اتفاقيات التسمية، الملفات موجودة، التسجيل مكتمل.

11 فحصاً استشارياً يجب أن تجتاز: معايير البرمجة، README موجود، التعاون موثَّق، التبعيات مثبَّتة.

فكر فيه كـ linter لفريق عملائك. يكتشف المشاكل الهيكلية قبل أن تصبح مشاكل وقت التشغيل.

## يعمل مع كل شيء (أو بدون شيء)

Squads محايد للـ framework. استخدمه منفرداً. استخدمه مع oh-my-claudecode أو GSD أو BMad Method أو أي framework تنسيق تفضله.

| الـ Framework | طريقة التكامل |
|-----------|------------------|
| Standalone | يعمل بمفرده، بدون تبعيات |
| oh-my-claudecode | تنسيق multi-squad عبر `team` و `ralph` و `autopilot` |
| GSD | Squads كمنفِّذين للمراحل عبر `execute-phase` |
| BMad Method | متوافق كمزوِّد squad في pipeline الـ BMad |
| Custom | أي شيء يستخدم Claude Code slash commands |

الـ framework يتولى التنسيق. Squads يتولى هيكل الفريق. فصل نظيف.

## لمن هذا

المطورون المستقلون الذين يريدون فرق ذكاء اصطناعي منظمة بدون شراء منصة.

الفرق الصغيرة التي تحتاج workflows قابلة للتكرار يمكن التحكم في إصداراتها.

الأشخاص الذين يبنون مع Claude Code وملّوا من فوضى "عميل واحد يفعل كل شيء".

كل من نظر إلى workflow الذكاء الاصطناعي الخاص به وفكر "هذا يحتاج إلى هيكل حقيقي."

## لمن هذا ليس

الأشخاص الذين يريدون منشئ عملاء بالسحب والإفلات المرئي. هذه ملفات markdown في مجلدات. إذا كنت تحتاج واجهة رسومية، ابحث في مكان آخر.

الأشخاص الذين لا يستخدمون Claude Code. Squads هي skill لـ Claude Code. تعمل داخل Claude Code. هذه هي الصفقة.

---

## يعمل على كل أنظمة الذكاء الاصطناعي للبرمجة. ليس فقط Claude Code.

هذه النقطة يفوتها كثيرون عند أول نظرة على Squads.

Squad هو مجلد يحتوي على ملفات markdown. skill هو معيار تفهمه كل أنظمة الذكاء الاصطناعي الكبرى للبرمجة. Claude Code، وCodex، وAntigravity، وGemini CLI — كلها تتحدث نفس اللغة: skills.

ثبِّت skill الـ Squads على Claude Code. تعمل squads الخاصة بك. ثبِّتها على Codex. نفس الـ squads، نفس العملاء، نفس الـ workflows. انتقل إلى Antigravity الشهر القادم. كل شيء لا يزال يعمل. مجلد `squads/` لا يهتم بأي نظام ذكاء اصطناعي يقرأه. العملاء هم markdown. المهام هي markdown. الـ workflows هي YAML. صيغ عالمية.

لست مقيداً بمورِّد واحد. لست تعيد كتابة تعريفات العملاء في كل مرة تغيِّر فيها الأدوات. ابنِ squads الخاصة بك مرة واحدة. شغِّلها في أي مكان.

هذه قابلية نقل حقيقية. ليست "يعمل على منصتنا" قابلية نقل. قابلية نقل تعمل على كل منصة.

## هناك سوق موجود بالفعل

لست مضطراً لبناء كل squad من الصفر.

[squads.sh](https://squads.sh) سوق ينشر فيه الناس squads الخاصة بهم. مجاني أو مدفوع — القرار لك. بنيت content-pipeline squad يعمل فعلاً؟ انشره. صنعت code-review squad يكتشف bugs تفوّت على أدوات أخرى؟ بِعه بما تريد. تحتاج squad لتدقيق SEO أو صفحات هبوط أو تحليل بيانات؟ على الأغلب أحد صنعه ووضعه هناك.

تصفَّح ما هو موجود. خذ ما يعمل. عدِّله. أو ابنِ من الصفر ودع الآخرين يستخدمونه.

في كل الأحوال، لن تبدأ وحيداً: [squads.sh](https://squads.sh)

## العرض في 30 ثانية

الآن لديك عميل ذكاء اصطناعي واحد يفعل كل شيء. بدون حدود. بدون تخصص. بدون أنماط تعاون. بدون تحقق.

Squads يمنحك فرقاً منظمة. كل عميل له وظيفة. كل مهمة لها شروط. كل workflow يحدِّد التعاون. كل شيء يعيش في مستودعك كـ markdown قابل للنقل. صفر تلوث للسياق. صفر تبعيات خارجية. صفر تكلفة.

ثبِّت في 30 ثانية. أنشئ أول squad في 60 ثانية. ابدأ الشحن مع فريق حقيقي بدلاً من عميل واحد يتظاهر.

```bash
cp -r squads-skill/ .claude/skills/squads/
mkdir -p squads/
/squads *create-squad my-first-squad
```

اذهب وابنِ فريقك. شحناً موفقاً.
